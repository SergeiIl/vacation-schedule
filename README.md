# План разработки приложения "Планировщик отпусков подразделения"

---

## **Цели приложения**:
1. Создать браузерное приложение для планирования отпусков с сохранением данных в локальном хранилище (IndexedDB).
2. Реализовать удобный интерфейс для ввода и редактирования отпусков сотрудников с визуализацией в виде диаграммы Ганта.
3. Добавить поддержку Drag-and-Drop (DnD) для интерактивного редактирования.
4. Обеспечить возможность задавать особые даты — государственные праздники и "запретные периоды", во время которых отпуск невозможен.
5. Реализовать функции экспорта и импорта данных в форматах JSON и CSV.
6. Использовать современные технологии и библиотеки для быстродействия, поддерживать Масштабирование данных на графике по дням/неделям/месяцам.

---

## **Технологический стек**:
1. **Vite** — для сборки проекта.
2. **React** — построение интерфейса приложения.
3. **Typescript** — строгая типизация для надёжности кода.
4. **Zustand** — управление состоянием.
5. **Tailwind CSS** + **Shadcn UI** — стилизация интерфейса.
6. **IndexedDB** (через библиотеку `idb`) — для хранения данных о сотрудниках, отпусках, праздниках.
7. **date-fns** — работа с датами.
8. **Framer Motion** — анимации взаимодействий.

---

## **Основной функционал**:

### 1. **Управление сотрудниками**
- Полный список сотрудников с фильтром поиска, сортировкой.
- Форма для добавления/редактирования сотрудников:
  - **ФИО (обязательно)**.
  - **Основной отпуск**:
    - Массив интервалов `[дата начала, дата окончания]`.
  - **НРД (нерабочие дни):**
    - `null` или заданный интервал `[дата начала, дата окончания]`.
- Автоматическая проверка:
  - Пересечение отпусков одного сотрудника.
  - Проверка на "запретные периоды".

---

### 2. **Диаграмма Ганта**
- Визуализация отпусков сотрудников в виде интерактивной диаграммы.
- Особенности:
  - Основной отпуск и НРД имеют разные цвета.
  - Особые даты (праздники, запретные периоды) отображаются на диаграмме:
    - Праздники — заливка фона.
    - Запретные периоды — выделение сеткой или другим цветом.
- Поддержка масштабирования:
  - **День, неделя, месяц**.
  - Адаптивный интерфейс для отображения больших объёмов данных.
- **Drag-and-Drop:**
  - Перетаскивание отпусков: изменение дат начала и окончания.
  - Перемещение интервалов отпуска (DnD).
  - Изменение размеров интервала (resizing).
  - Логика DnD с валидацией:
    - Нельзя заехать на "запретные периоды".
    - Проверка пересечения с другим отпуском.

---

### 3. **Особые даты и периоды**
- Раздел "Особые даты":
  - Редактор праздничных дней (например, общегосударственные выходные).
    - Поля: `название`, `дата или диапазон`.
  - Редактор "запретных периодов" (периоды, когда отпуск невозможен).
    - Поля: `название`, `начало`, `конец`.
- Отображение особых дат/периодов на диаграмме.
- Сохранение данных в IndexedDB.

---

### 4. **Экспорт и импорт данных**
- Экспорт:
  - Форматы: **CSV**, **JSON**.
  - Учитываются все данные о сотрудниках, отпусках и особых датах.
  - Пример структуры:
    ```csv
    ФИО,Тип отпуска,Дата начала,Дата окончания,Общее количество дней
    Иванов И.И.,Основной отпуск,2026-07-01,2026-07-14,14
    ```
- Импорт:
  - Загрузка из JSON (например, для восстановления данных).
  - Обработка ошибок некорректного формата.
  
---

### 5. **Хранение данных в IndexedDB**
- Заменить `localStorage` на **IndexedDB**:
  - Хранилище:
    - `Employees` — данные сотрудников и их отпуска.
    - `SpecialDates` — государственные праздники, запретные периоды.
    - `Settings` — текущие настройки приложения.
- Использовать библиотеку `idb` для упрощённой работы.
- Данные автоматически загружаются из IndexedDB при старте приложения.

---

### 6. **Дополнительные функции**
- **Поиск и фильтры:**
  - Поиск сотрудников по ФИО.
  - Фильтры: по наличию НРД, текущим отпускам.
- Настройки интерфейса:
  - Темизация (поддержка разных цветовых схем через Shadcn UI).
  - Масштабирование диаграммы.
- Отчёты:
  - Сводка: общее количество отпускных дней на сотрудника или отдел.

## **Интерфейс и основные компоненты**

### Компоненты:
1. **Базовые:**
   - `EmployeeList` — список сотрудников.
   - `AddEmployeeModal` — форма редактирования/добавления сотрудника.
   - `SpecialDatesEditor` — редактирование праздничных дат/периодов.
   - `GanttChart` — диаграмма Ганта отпусков.
2. **Функциональные:**
   - `DnDHandler` — обёртка для реализации DnD.
   - `ExportImportPanel` — модуль экспорта/импорта.
   - `SettingsPanel` — интерфейс настроек отображения.
3. **Утилиты:**
   - `dateUtils.ts` — общая работа с датами.
   - `indexedDB.ts` — методы взаимодействия с IndexedDB.
   - `validation.ts` — валидация данных.

## **Этапы разработки**

### Этап 1. Создание проекта
1. Настроить проект с использованием Vite.
2. Настроить библиотеки: Zustand, Tailwind CSS, Shadcn UI, date-fns.

### Этап 2. Реализация логики сотрудников и отпуска
1. Создать состояние (Zustand) для сотрудников и отпусков.
2. Разработать интерфейс добавления/редактирования сотрудников.

### Этап 3. Диаграмма Ганта
1. Подключить базовую корректную визуализацию отпусков.
2. Добавить поддержку DnD.
3. Реализовать масштабирование и особенности визуализации.

### Этап 4. Особые даты
1. Добавить редактор "особых дат".
2. Реализовать покрытие особых дат на диаграмме.

### Этап 5. Хранение данных в IndexedDB
1. Настроить IndexedDB для синхронизации состояния приложения.
2. Реализовать логику восстановления данных при перезагрузке.

### Этап 6. Экспорт/импорт данных
1. Реализовать экспорт JSON и CSV.
2. Импорт данных, обработка ошибок.
