Создай git коммит по конвенции проекта.

## Шаги

1. Выполни параллельно:
   - `git status` — список изменённых файлов
   - `git diff` — unstaged изменения
   - `git diff --staged` — staged изменения
   - `git log --oneline -10` — последние коммиты для контекста стиля

2. Определи **тип** коммита:
   - `feat` — новая функциональность
   - `fix` — исправление бага
   - `refactor` — рефакторинг без изменения поведения
   - `style` — форматирование, без изменения логики
   - `docs` — документация
   - `test` — тесты
   - `chore` — зависимости, конфигурация, сборка

3. Сформулируй сообщение **на русском языке**: `<тип>: <описание>`
   - Описание — краткое, в настоящем времени («добавление», «исправление», «вынос»)
   - Если изменения касаются нескольких несвязанных областей — предложи разбить на несколько коммитов

4. Добавь в индекс только релевантные файлы:
   - НЕ используй `git add -A` или `git add .`
   - НЕ добавляй `.env`, `.env.*`, файлы с секретами, `node_modules`
   - Добавляй по имени: `git add apps/backend/src/...`

5. Создай коммит через HEREDOC:
   ```
   git commit -m "$(cat <<'EOF'
   <тип>: <описание на русском>
   EOF
   )"
   ```

6. После коммита выполни `git status` для подтверждения успеха.

## Правила

- Описание ВСЕГДА на русском
- Не коммить secrets, не пропускай pre-commit хуки (`--no-verify` запрещён)
- Если pre-commit хук упал — исправь проблему и создай **новый** коммит (не amend)
- Co-Authored-By не нужен
